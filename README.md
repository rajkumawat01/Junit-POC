# Junit-POC
This repo is made to learn and implement Junit.

We use Junit to perform test driven development.

## Test Driven Development

#### Rules
1. Test the expected outcome of an example

2. Don't judge pre-design...  let your test drive it.

3. Write the minimum code required to get your tests to pass.

4. Always remember - RED -> GREEN -> REFACTOR

## Annotations

#### @Test
- Used to mark a method as a test case so the testing framework knows this need to be executed as part of unit testing.
- Method must be: void
